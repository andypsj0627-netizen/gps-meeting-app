# GPS Meeting App

모바일 GPS를 활용한 셀프 소개팅 앱. 근처에 있는 사람들과 자연스럽게 연결해주는 위치 기반 매칭 서비스.

- **앱 이름**: GPS Meeting App
- **기술 스택**: Flutter (크로스플랫폼 — iOS / Android)
- **패키지명**: `gps_meeting_app`
- **GitHub 레포**: `gps-meeting-app`

---

## 핵심 컨셉 (2026-07-14 확정)

앱의 두 축:
1. **실시간 조우** — 내 위치를 지도에 표시하고, 근처 사람과 조우하면 프로필이 단계적으로 열리고 호감을 표시할 수 있다
2. **시공간 겹침 등급** — 같은 시간대(1시간 버킷)에 같은 장소(150m 셀)를 지나친 사람은 "겹침"으로 기록되고, 자주 겹칠수록 등급이 올라 프로필이 더 열린다

두 축은 **하나의 조우 원장**을 공유한다(이벤트 소스만 둘). 실시간 조우는 등급을 크게 올리는 희소한 특별 이벤트("운명")다.

원칙:
- **어느 쪽이든 대화는 서로 호감을 표시했을 때만 열린다** (등급이 최고여도 채팅은 상호 호감이 조건 — 안전 게이트)
- "셀프 소개팅" — 앱이 강제 매칭하는 게 아니라 사용자가 주도적으로 탐색
- 상세 설계: [`docs/encounter-tiers.md`](./docs/encounter-tiers.md)

## 위치 데이터 모델 (2026-07-14 확정 — Phase 3 설계 기준)

Phase 3에서 `NearbyUsersService`의 Firestore 구현체를 만들 때 아래를 따른다.
나중에 바꾸면 스키마 마이그레이션 + 룰 재작성 비용이 크므로 처음부터 이 구조로 간다.

1. **위치는 프로필과 분리**: `users/{uid}`(프로필)와 별도로 `presence/{uid}`(현재 위치 + updatedAt + 공개 여부).
   갱신 주기(초 단위 vs 월 단위)와 읽기 권한이 달라서 문서를 분리해야 룰과 리스너가 깔끔하다.
2. **3층위 분리 저장**: geohash 셀(근처 쿼리용, 정밀도 6~7) / 퍼징 좌표(지도 표시용, 100~200m 격자 스냅) / 정밀 좌표(조우 판정용).
   조우 판정은 당분간 클라이언트가 하되(정밀 좌표 근처 읽기 허용), 출시 전 서버 판정으로 전환한다 —
   스키마를 미리 분리해두면 전환 시 보안 룰만 잠그면 되고 마이그레이션이 없다.
3. **근처 검색은 geohash 접두사 쿼리** (geoflutterfire_plus 등 표준 패턴).
4. **쓰기 주기**: 이동 중 10~15초 또는 30~50m마다, 정지 시 중단 (Firestore 쓰기 과금이 서버 비용을 결정).
   `updatedAt`이 오래된(수 분) presence는 지도에서 숨긴다 — 유령 마커 방지.
5. **위치 이력**(궤적 교차 조우용)은 정밀 좌표가 아니라 **geohash 셀 + 시간 버킷**만 저장, TTL 30일 자동 삭제.
